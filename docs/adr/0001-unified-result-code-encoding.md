# 0001-unified-result-code-encoding

三种接口（签名0x11、验签+签名0x13、验签0x15）统一使用一份结果码表，编码错开确保每个数值全局唯一含义：1/2专用于验签身份判定，3-6专用于验签失败，7-9专用于签名失败，≥10其他错误。0x12验签步骤仅result=0视为通过（result=1/2虽验签有效但不执行签名）。result=2的判定基于公钥比较，与输入id值无关，优先级高于result=1。

## Considered Options

1. **语义差异**：同一数值在不同接口含义不同（result=1在0x11为失败、0x13/0x15为验签通过），靠接口类型区分
2. **编码错开**：签名失败码从7开始，1/2留给验签身份判定，每个数值全局唯一含义
3. **拆分字段**：用不同JSON字段名消除歧义，但改动协议结构

## Consequences

- 客户端无需按接口类型切换解读逻辑，每个result值含义固定
- 0x13返回result=1/2时signed_data和id字段为空字符串，表示未执行签名步骤
- result=2检测证书身份冲突（公钥相同），是安全告警而非验签失败
